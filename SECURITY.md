# Security Policy

## Reporting a vulnerability

Please **do not** open a public issue for security vulnerabilities.

Report them privately using GitHub's
[private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability)
on this repository's **Security** tab.

Please include: a description, steps to reproduce, affected package/version, and any potential
impact.

## Response

- We aim to **acknowledge within 24 hours** and provide an initial assessment within 72 hours.
- Critical issues (API down, data corruption, security) follow the maintenance SLA: resolved or
  mitigated within 72 hours.
- We will keep you informed of progress and credit you in the advisory unless you prefer to
  remain anonymous.

## Scope

This policy covers the packages in this repository and the public RFP Hub API. The dataset is
public (CC0) by design; please report only genuine security issues, not the public availability
of the open data.
